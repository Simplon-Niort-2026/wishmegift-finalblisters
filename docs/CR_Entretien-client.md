# Compte-rendu d'entretien

## Pitch
En raison de l'_enshitification_ de Pinterest avec l'IA notamment, il souhaite proposer une alternative et voir si le projet est viable ou pas.

## Clientèle cible
Le client type est plutôt envisagé comme ayant des moyens de classe moyenne. Il est possible d'anticiper des montants plus importants pour convenir à une clientèle avec des moyens plus conséquents.

## Précisions sur certains aspects
- la date de l'évènement sert aussi de date butoir. C'est-à-dire qu'un utilisateur ne peut plus faire de réservations une fois la date passée,
- une description peut être vide,
- un lien est une URL qui peut autant pointer vers une boutique en ligne ou vers un site descriptif,
- le cadeau a un "statut" car il doit être indisponible s'il a été réservé par un autre utilisateur,
- un visiteur peut consulter une liste mais il ne peut pas agir dessus (il ne peut pas réserver par exemple),
- un utilisateur peut avoir plusieurs listes,
- un utilisateur peut avoir plusieurs listes sur le même thème,
- nous ne sommes pas concerné pas le front-end.

## Attendu du POC
- pouvoir partager des listes entre des utilisateurs factices pour mettre à l'épreuve,
- fournir une API à consommer pour le front-end.

## Élargissement envisageable du POC
- créer des listes pour une tierce personne,
- gestion du mail d'invitation à consulter.
